发表于 2022-06-04

设计模式代表有经验的面向对象软件开发人员使用的最佳实践。设计模式是软件开发人员在软件开发过程中面临的一般问题的解决方案。

设计模式的类型

3类23种

编号 模式和说明
1 创造模式 –创建对象而隐藏创建逻辑的方法，不是直接使用new来实例化对象
2 结构模式 –设计类和对象组成，继承概念用于组成接口并定义方法以获得新功能
3 行为模式 –侧重于对象之间的通信
4 J2EE模式 –设计表示层 由Sun Java Center表示

基本术语##

一、基本术语

数据**：基本值或基本值的集合

组项：具有从属数据项的数据项称为组项

记录：各种数据项的集合

文件：一种类型实体的各种记录的集合

属性和实体：实体表示某些对象的类，包含多个属性

字段：表示实体属性的基本信息单元

二、为什么需要数据结构##

随着应用程序越来越复杂，可能出现以下问题：

处理器速度：庞大的数据，处理器可能无法处理

数据搜索： 假设商店的库存大小是100860个商品，如果应用程序需要搜索某一特定商品，则需要遍历100860次，搜索过程非常慢

大量请求： 成千上万的用户同时在web服务器上搜索数据，导致服务器有大量请求

四、数据结构算法

算法是用于解决特定问题的明确定义的步骤的过程。 算法是有限的逻辑或指令集，它是为了完成某个预定义的任务而编写的。 它不是完整的程序或代码，它只是问题的解决方案(逻辑)，可以使用流程图或伪代码表示为非正式描述。

算法类别：

  • 插入：

时间衡量

  • 时间复杂度：开始到完成时间
  • 空间复杂度：算法执行过程中所需要的 内存空间

每个算法必须具有：

  • 规范：计算过程的描述
  • 前提条件：输入条件
  • 算法的主体：一系列清晰明确的指令
  • 后置条件：输出条件

算法特征：

  • 输入
  • 输出
  • 可行性：有限步数
  • 独立性
  • 明确无误
